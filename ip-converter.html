Understanding IPv4 Address Conversion

Every device connected to the internet uses an Internet Protocol (IP) address to uniquely identify itself. The most common format is IPv4, which consists of 32 bits, divided into four 8-bit sections called **octets**.

Why is Conversion Necessary?

Although humans read IP addresses in dotted-decimal format (e.g., 192.168.1.1), networking hardware (routers, switches) processes them as **binary** streams. The conversion to binary is fundamental to understanding subnetting, network masks, and routing protocols. The Decimal to Binary Rule: Each octet must be converted independently. Because an octet is 8 bits, the minimum decimal value is 0 ($00000000_2$) and the maximum is 255 ($11111111_2$). The conversion involves ensuring the binary representation is always padded with leading zeros to maintain exactly 8 bits.

Hexadecimal IP Addresses

Hexadecimal (Base 16) is a compact way to represent the 32-bit address. Each 8-bit octet is represented by two hexadecimal digits (00 to FF). Hex is often used in hardware documentation or when writing firewall rules due to its brevity compared to the 32-digit binary string.
